# Portfolio — Shrishti Sharma

A personal portfolio site built with **Laravel** and **Tailwind CSS**, featuring an
animated front end and a small admin area for managing showcased work.

## Features

- **Animated landing page** — hero, scroll, and reveal animations powered by GSAP and
  Lenis smooth scrolling.
- **Work showcase** — projects rendered from the database, ordered by a sortable field.
- **Admin section** — create / edit / delete work entries, restricted to the site owner
  via an authorization gate. Images are uploaded to storage with mime/size validation.
- **Contact form** — sends email on submission, with layered spam/abuse protection:
  honeypot field, time-based token, per-IP rate limiting, and URL/pattern heuristics.
- **Rate-limited login** — 5 attempts per email + IP before lockout.

## Tech stack

| Area      | Technology                          |
|-----------|-------------------------------------|
| Framework | Laravel 12 (PHP 8.2+)               |
| Frontend  | Blade, Tailwind CSS 3               |
| Bundler   | Vite (`laravel-vite-plugin`)        |
| Animation | GSAP, Lenis                         |
| Email     | Laravel Mail (Mailgun)              |

## Requirements

- PHP 8.2+
- Composer
- Node.js 18+ and npm

## Local setup

```bash
# 1. Install dependencies
composer install
npm install

# 2. Environment
cp .env.example .env
php artisan key:generate
# then fill in DB and MAIL_* credentials in .env

# 3. Database
php artisan migrate --seed

# 4. Run
npm run dev          # Vite dev server (assets + HMR)
php artisan serve    # app at http://localhost:8000
```

> The admin gate is keyed to a single owner account. Create that user (or seed it), then
> sign in at `/login` to reach the admin area at `/admin/works`.

## Building assets

```bash
npm run dev      # development build + hot reload
npm run build    # production build → public/build/
```

## Deployment

`public/build/` is generated by Vite and is **not** committed, so the build step must run
wherever the app is deployed:

```bash
composer install --no-dev --optimize-autoloader
npm ci && npm run build
php artisan migrate --force
php artisan optimize        # cache config, routes, views
```

Ensure `.env` has `APP_ENV=production` and `APP_DEBUG=false`.
